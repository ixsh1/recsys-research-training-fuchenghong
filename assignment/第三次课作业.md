## 4. 模型架构

SASRec 模型结构（`sasrec_demo.py`）：

| 组件 | 说明 |
|---|---|
| Item Embedding | `Embedding(item_count+1, hidden_size)`, padding_idx=0 |
| Position Embedding | `Embedding(max_len+1, hidden_size)`, padding_idx=0 |
| SASRec Blocks | `num_blocks` 个 pre-norm causal self-attention + FFN 块 |
| Causal Mask | 上三角矩阵，禁止当前 token 关注未来 token |
| Final LayerNorm | 输出最终序列表征 |

课堂运行参数：

```text
hidden_size=64, num_heads=2, num_blocks=2, max_len=20
dropout=0.2, learning_rate=0.001, batch_size=32
```

## 5. 训练流程

训练使用二元交叉熵损失（BCEWithLogitsLoss），每个位置配对正样本和负样本：

1. **数据读取**: 加载 CSV，重映射 ID，划分 train/valid/test
2. **负采样**: 按位置独立采样用户未交互过的物品作为负样本
3. **序列截断**: 右对齐 + 左 zero-padding，保留最近 `max_len` 个交互
4. **训练循环**: 每个 epoch 遍历所有用户，计算正/负样本 logit 的 BCE loss
5. **验证**: 每个 epoch 结束后在全物品集合上评估 HR@K 和 NDCG@K
6. **Checkpoint**: 按验证集 (NDCG, HR) 字典序保留最佳模型

## 6. 关键 Shape

| 阶段 | 变量 | shape | 含义 |
|---|---:|---|---|
| 输入序列 | `sequences` | `(B, max_len)` | B 个用户的右对齐 item 序列 |
| 正样本 | `positives` | `(B, max_len)` | 每个位置的下一个真实 item |
| 负样本 | `negatives` | `(B, max_len)` | 每个位置的随机负样本 item |
| 嵌入输出 | `features` | `(B, max_len, hidden_size)` | 序列表征 |
| 正样本 logit | `positive_logits` | `(B, max_len)` | 每个位置的正样本分数 |
| 全物品打分 | `score_all_items` 输出 | `(B, item_count+1)` | 最终位置对全部物品的打分 |

## 8. 新增知识/概念

### SASRec 核心思想

- 使用 self-attention 机制对用户历史交互序列建模
- 通过 causal mask 保证时间因果性（当前步不能看到未来步）
- 通过 item embedding 和 position embedding 共同编码序列信息

### 序列推荐评估

- **HR@K (Hit Rate)**: 目标物品是否出现在推荐的 top-K 列表中
- **NDCG@K (Normalized Discounted Cumulative Gain)**: 考虑命中位置，排名越靠前得分越高
- **Leave-two-out**: 每用户最后两个交互分别用于验证和测试

### GPU 显存管理

- `--allocator-limit-mib` 限制 PyTorch 缓存分配器可用显存
- `--gpu-budget-mib` 作为 nvidia-smi 实际进程显存上限
- 程序在模型初始化、每个 epoch 结束和最终评估后采样显存并检查预算
