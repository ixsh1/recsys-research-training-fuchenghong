## 3. 关键 shape

| 阶段 | 变量 | shape | 含义 |
|---|---|---:|---|
| 全部输入 | `features` | `(240, 2)` | 240 个二维样本 |
| 全部标签 | `labels` | `(240,)` | 每个样本一个 0/1 标签 |
| 数据划分 | train / valid / test | `160 / 40 / 40` | 训练、验证、测试集样本数 |
| 一个 Batch | `batch_features` | `(16, 2)` | 一次输入模型的 16 个二维样本 |
| 一个 Batch | `batch_labels` | `(16,)` | 与 batch 样本对应的标签 |
| 模型输出 | `logits` | `(16,)` | 每个样本一个未过 sigmoid 的原始分数 |
| 批次损失 | `loss` | `()` | 一个标量，用于反向传播 |

代码中将模型和 batch 数据移动到 GPU：

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = BinaryClassifier().to(device)
batch_features = batch_features.to(device)
batch_labels = batch_labels.to(device)
```

## 5. 概念说明

**Tensor**：PyTorch 中保存数值的多维数组，带有 shape、dtype 和 device。本实验的 `features` 是 `(240, 2)` 的 tensor，训练时 batch 会被移动到 `cuda`。

**Batch**：一次送进模型的一小组样本。本实验 batch size 为 16，所以一个完整 batch 的输入是 `(16, 2)`，标签是 `(16,)`。

**Epoch**：完整遍历一遍训练集。本实验训练集 160 条样本，batch size 16，所以每个 epoch 有 10 次参数更新。

**Logit**：模型输出的原始分数，还不是概率。`logit > 0` 时 sigmoid 后概率大于 0.5，更倾向预测为 1。

**Loss**：衡量预测和真实标签差距的标量。这里使用 `BCEWithLogitsLoss`，它把 sigmoid 和二分类交叉熵合在一起，数值更稳定。
